Dotfiles

On a Unix system programs are most of the time configured in two different ways, using the command-line or by editing a configuration file. Those configuration files are most of the time have located somewhere in the home directory (~). By default, most of those files are hidden because they start with the dot (.) character, hence the name dotfiles.

Because these files are important to some people, they need to be backed up somewhere. This is most of the time done using a Git repository either on GitHub or somewhere else. The repository then allows them to reinstall a system and restore their configuration without the need of manually configuring them. Well this is exactly what I've done and this repository is for.

The Git repository contains a bunch of configuration files that I use to configure the programs I use (such as Git and Vim). It also contains a collection of scripts that I use to automate certain tasks on my system and maybe a few small command-line based games. But that is not the only purpose of this repository, I also use this repository to learn more about Linux, Git and automating tasks.

Here is a small list of what you can find in this repository:

  • Bash configuration files
  • Termite configuration file
  • Git configuration files
  • Vim configuration files
  • Spicetify configuration files
  • Gnome-Shell configuration files
  • Wallpapers
  • A fancy installer
  • And a bunch more

Note: Most of these files are specifically for Fedora Linux. Check out the repository for more information.
