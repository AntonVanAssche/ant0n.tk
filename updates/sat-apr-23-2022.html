BashPass version 2.0 released!

- Sat Apr 23 2022

BashPass version 2.0, this release focusses primarily new features such as the ability to synchronize passwords between different devices. But it also focusses on the configuration of the program.

A new configuration format has been implemented, making it easier to configure BashPass. Because of this new format, the installer is split into separate scripts. This change makes it easier to update your current configuration to the new format.

Something that's also new is a default password length (14) to generate a new password, of course you can still enter another number to generate a longer or shorter password.

From now on a GPG key ID has to be added to the config file, this allows you to encrypt your password with the same key every time. This is useful if you have more than one GPG key on your system. It also ensures that you only need to export or import one key when using the `--sync` option.

There was also a fix to the compatibility of BashPass on BSD systems such as FreeBSD and OpenBSD.

There was also a big change in the documentation, which is now located at the wiki page on GitHub.

For a more complete list of changes, see Bashpass 2.0's Pull Request or the release notes on GitHub.
